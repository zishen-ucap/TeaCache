# TeaCache4Cosmos

[TeaCache](https://github.com/LiewFeng/TeaCache) can speedup [Wan2.1](https://github.com/Wan-Video/Wan2.1.git) 3.1x without much visual quality degradation, in a training-free manner. The following video shows the results generated by TeaCache-Cosmos with various `rel_l1_thresh` values: 0 (original), 0.1 (2.0x speedup), and 0.2 (3.1x speedup). Additionally, the image-to-video (i2v) results are also demonstrated, with the following speedups: 0.1 (1.6x speedup), and 0.2 (2.5x speedup). Due to the nature of the TeaCache algorithm, it requires additional space to cache intermediate latent representations. This extra memory usage leads to Out of Memory (OOM) issues when running the I2V 720p model, as it already nearly maxes out the H100 GPU's capacity. As a result, the I2V mode currently only supports 480p resolution to ensure stable performance.

## Usage

Follow [Wan2.1](https://github.com/Wan-Video/Wan2.1.git) to clone the repo and finish the installation, then you can modify the `rel_l1_thresh` to obtain your desired trade-off between latency and visul quality, and change the `checkpoint_dir`, `prompt`, `input_image_or_video_path` to customize your video.

You need to copy the running script to the Cosmos project folder:

```bash
cd Wan2.1
cp *.py /path/to/Cosmos/
```

For T2V inference, you can use the following command:

```bash
cd /path/to/Wan2.1/

python3 teacache_sample.py \
    --size 1280*720 \
    ----ckpt_dir Wan2.1-T2V-14B \
    --prompt "Two anthropomorphic cats in comfy boxing gear and bright gloves fight intensely on a spotlighted stage." \
    --base_seed 42 \
    --offload_model True \
    --t5_cpu \
    --rel_l1_thresh 0.2 \
```

For I2V inference, you can use the following command:

```bash
cd /path/to/Wan2.1/

python3 teacache_sample.py \
    --task i2v-14B \
    --size 832*480 \
    --ckpt_dir Wan2.1-I2V-14B-480P \
    --image 2.png \
    --prompt, "A girl gazed at the camera and smiled, her hair drifting in the wind." \
    --base_seed 42 \
    --rel_l1_thresh 0.2 \
```
